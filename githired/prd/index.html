Section 01

The overview

GitHired is a web application that streamlines the job-application process. Users supply their career history (CV) and a sample cover letter once, then manage individual job opportunities on a Trello-style Kanban board. For each posting, the platform scrapes, normalises, and tailors — producing a resume and cover letter whose every inclusion and exclusion is annotated against the requirement it answers. Everything exports to ATS-friendly PDF.

Section 02

The problem

Applying to jobs is repetitive and time-consuming. Candidates re-format the same information dozens of times, miss keywords, and fail to align their materials with the specific posting. Existing tools either automate too little (spreadsheets, Notion boards) or too much (one-click apply spam). There is no single tool that combines pipeline tracking and intelligent, explainable document generation.

"I'm on application number 87. I have no idea what I sent to which company, and every cover letter looks like the last one with the company name swapped in."
Section 03

Target users

Persona Description

Passive job seeker Casually tracking a handful of roles. Values organisation and quick generation when ready to apply.
Career switcher Needs help surfacing transferable skills. Benefits most from the AI's relevance-mapping and transparency report.

Section 05

Core features

5.1 — User profile & CV intake

Onboarding

A structured form (or paste / upload) captures contact info, summary, work experience, education, skills, certifications, and projects. The user provides at least one sample cover letter so the AI can learn tone, structure, and voice.

  • Profile is the single source of truth — every future generation reads from it.
  • Edits propagate forward; locked snapshots on previously-submitted applications stay intact.
  • CV editor supports freeform text sections for unconventional careers.

5.2 — The Kanban board

Trello-style

A horizontally-scrollable set of columns; cards drag between columns or reorder within them. Columns are user-editable — add, rename, reorder, remove.

Column Purpose
BacklogNewly created — not yet prepared.
PendingResume & cover letter being drafted or edited.
AppliedApplication has been submitted.
InterviewingActively in the interview process.
CompletedOffer accepted, rejected, or withdrawn.

Card appearance

  • Job title & company.
  • Location (remote / on-site / hybrid).
  • Salary range (if available).
  • Date added · column-coloured label.
  • Interview-progress badge once an interview list exists (e.g. 2 / 5 done).

5.2.1 — Create application flow

URL → card
  1. User clicks + Create Application (always visible above the board).
  2. Modal opens with a single required field: job posting URL.
  3. System scrapes & normalises the posting and auto-populates the card.
  4. Card lands at the top of Backlog.
  5. On scrape failure: falls back to a manual form pre-filled with whatever was extractable.

5.2.2 — Job Detail page

Full-page

Clicking a card opens a full-page detail view — not a modal. The page has a left side-nav and an action bar pinned at the bottom.

5.2.4 — Apply & confirm workflow

Snapshot lock
  1. Click Apply in the persistent action bar.
  2. Final Review screen — read-only previews of resume & cover letter, plus Copy and Download PDF buttons for each.
  3. Click Mark as Applied.
  4. Confirmation dialog: "Have you submitted your application for [Job] at [Company]?"
  5. On Yes: card moves to Applied; applied_at stamped; resume + cover letter locked as a versioned snapshot.
  6. On Not Yet: dialog closes; user stays on the review screen.

5.2.5 — Interview tab

TODO checklist

A list of interview subjects with title, free-text notes (date, interviewer, prep links), a checkbox, and a created-at timestamp. Drag-to-reorder. Completing strikes through and moves to the bottom. A small 2 / 5 progress fraction appears at the top of the tab and on the Kanban card.

Section 06 · The differentiator

The Tool tab — AI assistant

The Tool tab is where the product earns its $10/mo. It contains two sub-sections — the Resume Builder and the Cover Letter Builder — both of which annotate every choice against the posting it answers and the CV detail it sourced.

Resume builder

  1. AI generates a tailored resume using the matching algorithm in §08.
  2. Result is displayed as an editable document — add, remove, reword any line before finalising.
  3. Each included CV entry shows the posting requirement it answers as an inline annotation, plus a relevance tag (Hard Requirement, Responsibility, Nice-to-Have, Bonus Strength).
  4. Annotations are optional — an entry may be included purely because it's a strong achievement, even without a posting match.
  5. Excluded entries appear in a collapsible "Not Included" section with a brief reason — manually pullable.

Cover letter builder

  1. AI generates a tailored cover letter using the user's sample for tone, structure, voice.
  2. Result is an editable rich-text document.
  3. Key phrases or paragraphs annotate with: the CV detail that sourced the claim, the posting text it addresses, and a short rationale.
  4. User edits freely; annotations update or disappear as underlying text changes.

Regenerate

Both builders offer a Regenerate button that re-runs the AI pipeline while preserving any manual edits the user has pinned or locked.

Section 07

Job posting scraper & normaliser

  1. Input — user pastes a URL.
  2. Scrape — backend fetches the page; common job boards covered (LinkedIn, Indeed, Greenhouse, Lever, Workday).
  3. Normalise — AI parses raw content into a structured schema (title, company, location, salary, requirements by category, responsibilities, nice-to-haves, benefits, raw text).
  4. Fallback — if scraping fails (auth-walled page), user pastes the posting text manually.

Section 08

AI pipeline & matching algorithm

The matching engine takes the user's CV, the user's sample cover letter, and the normalised posting — and produces tailored documents plus a transparency report for each.

└────────────┘     │   AI Matching  │◀────│    Posting       │
┌────────────┐     │    Engine      │     └──────────────────┘
│  Sample    │────▶│                │

                  │                 │

Priority tiers

  1. Tier 1 — Hard requirements (e.g. "Must have 5+ years React") → Highest weight. Surface the most relevant CV bullets.
  2. Tier 2 — Responsibilities (e.g. "You will lead a team of 5") → Medium weight. Surface leadership / collaboration evidence.
  3. Tier 3 — Nice-to-haves (e.g. "GraphQL is a plus") → Lower weight. Include if space allows.
  4. Tier 4 — Unmatched CV strengths → Include standout achievements even without a direct posting match, space permitting.
Key rule: the system never fabricates experience. If no CV entry matches a requirement, that requirement is noted as unmatched in the report — it is not filled with invented content.

Transparency report — resume

CV entry Status Matched requirement Reason
"Led migration to React 18 at Foo Inc." ✓ Included "5+ years of experience with React" Direct skill match — demonstrates hands-on React leadership.
"Managed office supply budget" — Excluded — No relevance to any listed requirement.

Transparency report — cover letter

Detail used CV source Matched posting text Why selected
"Reduced page load by 40%" Work Experience → Foo Inc. "Performance-minded engineering culture" Quantifiable impact aligned with stated company value.
Section 09

PDF export

  • Resume PDF — professionally formatted, ATS-friendly, one-click download.
  • Cover letter PDF — matching visual style, addressed to the correct company & role.
  • Combined PDF — option to download both as a single document.
  • Templates are minimal and clean by default; template selection is a v2 question.

Data model

  │    ├─ full_name, contact_info, summary

  └─ JobCard
       ├─ id, url, raw_html, normalised_json
       ├─ status, column_id, position
       ├─ created_at, applied_at?, completed_at?
       ├─ Comments[]
       ├─ InterviewItems[]   (title, description, is_complete, position)
       ├─ GeneratedResume?
       │    ├─ content_json, pdf_url, is_locked

            ├─ content_text, pdf_url, is_locked

Section 17

Risks & mitigations

Job-board anti-scraping
medium

LinkedIn, Indeed, and Workday actively block automated fetches; ToS issues plus technical blockers.

Mitigation — manual paste fallback always available; rotate user-agents; prefer official APIs (Greenhouse, Lever) where they exist.
AI hallucination
high

An LLM can confidently invent experience the user does not have — a single bad submission destroys trust.

Mitigation — strict prompts that ban invention; transparency report exposes every claim; user verifies before locking the snapshot.
LLM cost at scale
medium

Per-generation OpenAI calls compound quickly. A Premium user generating 20 applications/month materially affects margin.

Mitigation — cache normalised postings; allow user-provided API keys; tiered pricing with generation caps on Free.
PDF formatting drift
low

Recruiter ATS systems reject documents with unusual layout. Unprofessional output torches conversion.

Mitigation — constrained templates (single-column, system fonts only); cross-browser/device tests; ATS-validation pass before release.
User data sensitivity
high

Resumes contain detailed PII. A breach is reputation-fatal in a market where trust is the only moat.

Mitigation — encrypted at rest & in transit; never share resume content with third parties; one-click export & delete-account; published incident-response policy.
Solo-founder bandwidth
medium

Support, ops, growth, and engineering on one operator. Burnout is the failure mode.

Mitigation — hard scope ceiling at MVP v1.0; ship-fast iteration on Reddit/LinkedIn feedback; outsource support emails before scaling paid channels.

Future considerations

  • Browser extension — capture postings without leaving LinkedIn / Indeed / Greenhouse. Differentiator vs. Teal, Huntr, Simplify.
  • Email tracking — connect inbox to auto-update card status on recruiter replies.
  • Interview prep — AI-generated question sets based on posting + user's resume.
  • Analytics — response rates by keyword, company, role type.
  • Team / cohort mode — career coaches manage boards for multiple clients.
  • Template marketplace — community-contributed resume & cover-letter templates.
